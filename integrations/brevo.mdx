This integration lets you **push your Mailkick email HTML** to **Brevo Transactional templates**.

## Prerequisites

- You must be a **workspace admin**.
- You need a **Brevo API key** for the Transactional/SMTP API (not the Marketing API).

## Step 1 — Find your Brevo API key

In Brevo, go to **SMTP & API**, then create/copy an **API key** (Transactional/SMTP).

## Step 2 — Add Brevo in Mailkick

1. Open **Settings → Integrations**
2. Select **Brevo**
3. Paste your API key
4. Click **Save changes**

<img
  style={{ borderRadius: '0.75rem' }}
  src="/app/doc/integration/integration_brevo.jpg"
  alt="Mailkick Brevo integration settings"
/>

<Tip>
If you see errors like <code>brevo_invalid_route_or_method</code>, the most common cause is using the wrong Brevo key type. Create a key in Brevo under <strong>SMTP &amp; API</strong>.
</Tip>

## Step 2.1 — (Optional) Select a default sender

Brevo requires a sender. In **Settings → Integrations → Brevo**:

1. Click **Fetch senders**
2. Pick a verified sender as your **Default sender**
3. Click **Save changes**

If no sender is configured, Mailkick will prompt you during sync.

## Step 3 — Sync an email

1. Open an email in the editor
2. Click **Export**
3. Click **Sync to Brevo**

After a successful sync, you’ll get a Brevo editor link like:

- `https://my.brevo.com/camp/template/<template_id>/message-setup`

## Common issues

- **“Save before syncing”**: the email needs a saved version before it can be synced.
- **Sender errors**:
  - `brevo_sender_missing`
  - `brevo_sender_invalid_or_inactive`
